Workflow

Everything visible, nothing theatrical.

Kiro keeps the archive workflow direct: choose a source, confirm the destination, check manifest status, then run the legacy engine without changing resource bytes behind your back.

01

Pack folders

Build a PAK from a source folder and write the matching TXT sidecar beside it.

02

Unpack safely

Detect sidecars before extraction and explain when original filenames are unavailable.

03

Inspect outputs

Inventory recovered names, typed unknowns, unknown binaries, and report paths without rewriting payloads.

Compatibility

Built around the legacy x86 archive engine.

x86 Windows Runs natively.
x64 Windows Runs through standard 32-bit app compatibility.
Windows on ARM Runs through x86 emulation. Native ARM64 is not included because the engine DLLs are x86.
