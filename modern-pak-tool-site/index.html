Core workflow

Built for the two jobs that matter.

01

Pack folders

Choose a source folder and the app suggests a sibling .pak path. The legacy engine writes the archive and matching .pak.txt sidecar.

02

Unpack archives

Select a PAK and output folder. Matching TXT sidecars are preferred so original paths are preserved during extraction.

03

Recover names honestly

When sidecars are missing, reference manifests can restore known paths. Otherwise the app keeps generated IDs and only adds safe inferred extensions.
